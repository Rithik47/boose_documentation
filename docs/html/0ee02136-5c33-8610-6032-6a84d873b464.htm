      BOOSEapp Namespace

      The BOOSEapp namespace contains all classes and components for the BOOSE (Basic Object-Oriented Simple Environment) application. This namespace provides a complete implementation of an educational programming environment with drawing capabilities.

      Remarks

      The BOOSEapp namespace is organized into several functional areas:

      • Core Engine – Interpreter, AppCanvas, AppVariable - Main execution and state management
      • Drawing Commands – AppCircle, AppMoveTo, AppRect, AppPenColour, AppWrite - Drawing operation implementations following the Command pattern
      • Language Parsers – AppInt, AppReal, AppBool, AppArray, AppIf, AppWhile, AppFor - Statement and control structure parsers
      • Data Structures – MethodDef, ParserException - Supporting data types and exceptions
      • User Interface – Form1, Program - Windows Forms-based UI for code editing and canvas display
      • Factory Components – AppCommandFactory, AppMethodParser - Factory and parser utilities

      Language Features Supported:

      • Variable types: int, real (double), boolean
      • Arrays: integer and real arrays with poke/peek operations
      • Control structures: if/else, while loops, for loops
      • User-defined methods with parameters and return values
      • Expression evaluation: arithmetic, boolean, and comparison operators
      • Drawing commands: moveto, drawto, circle, rectangle, triangle, write, pen color

      Architecture Overview:

      The application follows a layered architecture with clear separation of concerns:

      1. Presentation Layer: Windows Forms UI (Form1, Program)
      2. Core Engine Layer: Program interpreter, canvas manager, variable storage
      3. Command Layer: Drawing command implementations using Command pattern
      4. Parser Layer: Language syntax parsing and validation
      5. Data Model Layer: Shared data structures and exception types

      Design Patterns Used:

      • Command Pattern: Drawing commands encapsulate operations
      • Factory Pattern: AppCommandFactory creates command instances
      • Interpreter Pattern: Interpreter class parses and executes BOOSE language
      • Strategy Pattern: Expression evaluation strategies for numeric and boolean types

      Technical Stack:

      • Target Framework: .NET 10
      • Language: C# 14.0
      • UI Framework: Windows Forms
      • Graphics: System.Drawing (GDI+)
      • External Dependencies: BOOSE Framework (ICanvas, Command base classes)

      Typical Usage Example:

      C#
      // Create canvas and variable storage
var canvas = new AppCanvas(800, 600);
var vars = new AppVariable();

// Create interpreter
var interpreter = new Interpreter(canvas, vars);

// Execute BOOSE program
string program = @"
    int x = 100
    int y = 100
    pen 255,0,0
    moveto x,y
    circle 50
";
interpreter.RunProgram(program);

// Get rendered bitmap
var bitmap = canvas.getBitmap();

      Project Purpose:

      This application serves as an educational tool for teaching programming concepts through visual feedback. Students can write simple programs in the BOOSE language to create drawings, learn control structures, practice problem-solving, and see immediate visual results of their code. The system is designed to be beginner-friendly while supporting advanced features like recursion and user-defined methods.

      Quality Attributes:

      • Usability: Simple syntax designed for beginners
      • Reliability: Comprehensive error handling and validation
      • Maintainability: Well-documented code with XML comments
      • Extensibility: Modular design allows easy addition of new features
      • Testability: Unit tests cover core components and parsers

      Classes

      AppArray Static helper class for handling array declaration, poke (write), and peek (read) operations. Supports both integer and real (double) arrays with element-level access.
      AppBool Static helper class for handling boolean variable declaration and assignment statements. Supports parsing and processing of 'boolean' declarations and assignments to existing boolean variables.
      AppCanvas Application implementation of the BOOSE canvas. Provides drawing primitives used by the command implementations and exposes position and bitmap accessors. Implements the ICanvas interface for integration with the BOOSE command framework.
      AppCircle Command implementation to draw a circle centered on the current pen position. Inherits from CommandOneParameter to handle a single integer radius parameter.
      AppFor Static helper class for parsing FOR loop blocks from source code lines. Handles for/end for block structure with support for nested loops and optional step values.
      AppIf Static helper class for parsing IF conditional blocks from source code lines. Handles if/else/endif block structure with support for nested conditionals.
      AppInt Static helper class for handling integer variable declaration and assignment statements. Supports parsing and processing of 'int' declarations and assignments to existing int variables.
      AppMethodParser Parser for method/function-like declarations. Supports simple forms like: method name = expr end method or method int name int p1, int p2 name = ... end method
      AppMoveTo Command implementation to move the pen to a specified position without drawing. Inherits from CommandTwoParameters to handle X and Y coordinate parameters.
      AppPenColour Command implementation to set the pen colour using RGB components. Inherits from CommandThreeParameters to handle red, green, and blue color components.
      AppReal Static helper class for handling real (double) variable declaration and assignment statements. Supports parsing and processing of 'real' declarations and assignments to existing real variables.
      AppRect Command implementation to draw a rectangle at the current pen position. Inherits from CommandTwoParameters to handle width and height parameters.
      AppVariable Manages variable storage and expression evaluation for the BOOSE language interpreter. Supports integers, reals (doubles), booleans, arrays, and user-defined methods. Provides expression parsers for numeric and boolean expressions with operator precedence.
      AppWhile Static helper class for parsing WHILE loop blocks from source code lines. Handles while/end while block structure with support for nested loops.
      AppWrite Command implementation to write text or numeric output to the canvas. Inherits from CommandOneParameter to handle a single integer value parameter.
      Form1 Main application form hosting the BOOSE canvas and controls for running BOOSE programs. This version runs variable/real/int replacement in-app (no BOOSE Int/Real restrictions).
      Interpreter The main interpreter for executing BOOSE language programs. Parses and executes source code including variables, control structures, methods, and drawing commands.
      MethodDef Represents the definition of a user-defined method in the BOOSE language. Contains metadata about the method including its name, return type, parameters, and body.
      ParserException Exception type for parser/runtime errors in the BOOSE interpreter.
